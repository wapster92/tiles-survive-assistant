## Замечания для модели данных

- Один и тот же элемент может появляться в разные дни с тем же коэффициентом.
- Некоторые строки описывают не предмет, а действие: убийство зараженных, сбор ресурсов, покупка наборов, обучение войск.
- Для калькулятора лучше хранить отдельно `element`, `action`, `unit`, `points`, `days`, чтобы UI мог группировать правила по дням или по типам ресурсов.
