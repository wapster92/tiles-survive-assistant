# База данных

## Выбор

Для MVP используется SQLite. Пользователей ожидается мало, справочники небольшие, а основные операции — чтение правил и сохранение пользовательских запасов.

Текущий стек:

- `SQLite` — файл базы;
- `better-sqlite3` — драйвер;
- `Drizzle ORM` — схема и запросы.

## Расположение файла

Путь задается переменной окружения:

```text
SQLITE_DATABASE_PATH=./data/tails-survive.sqlite
```

В Docker Compose база хранится в volume:

```text
/data/tails-survive.sqlite
```

## Таблицы MVP

### elements

Справочник игровых элементов.

| Поле | Значение |
| --- | --- |
| `key` | стабильный код элемента |
| `name` | отображаемое название |
| `category` | категория для группировки |
| `unit` | единица измерения |
| `icon` | путь к SVG-иконке |
| `days` | дни события, где элемент встречается |
| `tier` | уровень разновидности элемента, если применимо |
| `color_name` | название цвета tier на русском |
| `color_hex` | цвет tier для интерфейса |

### event_rules

Правила начисления очков.

| Поле | Значение |
| --- | --- |
| `key` | стабильный код правила |
| `element_key` | ссылка на `elements.key` |
| `action` | текст действия |
| `points` | очки за действие |
| `days` | дни события, где правило активно |
| `sort_order` | порядок отображения |
| `calculator_visible` | доступно ли правило пользователю; внутренние коэффициенты опыта скрыты |

### users

Минимальная таблица аккаунтов.

| Поле | Значение |
| --- | --- |
| `id` | внутренний идентификатор |
| `username` | уникальное имя пользователя для входа |
| `game_nickname` | ник в игре; может содержать кириллицу |
| `nickname` | legacy-дубль `username` для старых SQLite-баз |
| `password_hash` | `scrypt`-хеш пароля |
| `created_at` | дата создания |

Пароль в открытом виде не хранится.

### user_element_values

Значения, которые пользователь ввел по элементам справочника.

| Поле | Значение |
| --- | --- |
| `user_id` | ссылка на пользователя |
| `element_key` | ссылка на `elements.key` |
| `amount` | текущее количество |
| `updated_at` | дата последнего обновления |

Первичный ключ составной: `user_id + element_key`.

### user_plan_items

Выбранные пользователем строки конструктора калькулятора.

| Поле | Значение |
| --- | --- |
| `user_id` | пользователь |
| `rule_key` | выбранное правило начисления |
| `amount` | количество в запасе |
| `planned_amount` | количество, которое планируется потратить |
| `universal_amount` | универсальные минуты на действие в выбранный день |
| `day` | выбранный день траты |
| `updated_at` | дата обновления |

Первичный ключ: `user_id + rule_key + day`. Одно правило можно распределить по нескольким доступным дням. Поле `amount` общее для строк одного правила, а сумма `planned_amount` не может превышать `amount`.

### user_training_plans

Параметры казармы и специализированные ускорения обучения.

| Поле | Значение |
| --- | --- |
| `enabled` | добавлена ли форма обучения в пользовательский план |
| `soldier_level` | уровень обучаемых солдат |
| `current_soldiers` | занято мест в гарнизоне |
| `garrison_limit` | общая вместимость гарнизона |
| `planned_soldiers` | планируемое количество новых солдат |
| `batch_limit` | максимальный размер одной очереди |
| `batch_hours` | часы обучения полной очереди |
| `batch_minutes` | минуты обучения полной очереди |
| `training_speedup_minutes` | специализированные минуты ускорения обучения |
| `universal_training_minutes` | универсальные минуты, выделенные на обучение |

## Seed

Справочники сидируются при старте backend из файлов:

- `backend/src/db/seed-data.js`
- `backend/src/db/schema.js`

Seed идемпотентный: существующие записи обновляются по ключу, новые добавляются.

Также при старте создается тестовый пользователь, если его еще нет:

| Имя пользователя | Пароль |
| --- | --- |
| `root` | `0000` |

## API

- `GET /api/dictionaries` — полный набор `elements` и `rules`;
- `GET /api/elements` — только элементы;
- `GET /api/rules` — все правила;
- `GET /api/rules?day=7` — правила конкретного дня;
- `GET /api/rules?elementKey=gear-scraps` — правила конкретного элемента.

### Пользователи

- `POST /api/auth/register` — регистрация по `username`, `password`, `gameNickname`;
- `POST /api/auth/login` — вход по `username` и `password`;
- `GET /api/me` — текущий пользователь по Bearer-токену.

### Значения пользователя

- `GET /api/me/element-values` — получить сохраненные значения;
- `PUT /api/me/element-values` — сохранить или обновить значения;
- `DELETE /api/me/element-values` — сбросить все значения;
- `DELETE /api/me/element-values/:elementKey` — удалить значение одного элемента.

### Конструктор калькулятора

- `GET /api/me/calculator` — получить план, ускорения и расчет;
- `PUT /api/me/plan-items` — заменить строки плана;
- `PUT /api/me/speedup-allocation` — сохранить общий пул универсальных ускорений; распределение хранится в строках плана;
- `PUT /api/me/training-plan` — сохранить параметры казармы и план обучения;
- `GET /api/me/calculation` — рассчитать сохраненный план;
- `DELETE /api/me/calculator` — сбросить все данные калькулятора.

Формат сохранения:

```json
{
  "values": [
    {
      "elementKey": "bolt-tier-2",
      "amount": 50
    }
  ]
}
```

Защищенные ручки используют заголовок:

```text
Authorization: Bearer <token>
```
