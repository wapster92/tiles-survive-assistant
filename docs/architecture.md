# Архитектура

## Общий подход

Проект разделен на два приложения:

- `frontend` — Vue-приложение с формой расчета и отображением результата.
- `backend` — Express API, где будут храниться правила расчета, коэффициенты, расписание и будущая бизнес-логика.

Такой раздел оставляет возможность отдельно разворачивать UI и API, а также позже добавить базу данных без переписывания фронта.

## Frontend-стек

- `Vue 3` + `Vite` — приложение и сборка;
- `Nuxt UI` — компоненты и дизайн-система в режиме plain Vue;
- `Tailwind CSS` — стили Nuxt UI и будущая тема проекта;
- `@vueuse/motion` — короткие функциональные анимации интерфейса;
- `axios` — единый HTTP-клиент для backend API;
- `vue-router` — маршрутизация и обязательная peer-зависимость Nuxt UI.

## API на старте

- `GET /health` — проверка доступности API.
- `GET /api/event-context` — краткий контекст события и стартовое расписание.
- `GET /api/dictionaries` — элементы и правила начисления очков из SQLite.
- `GET /api/elements` — справочник элементов.
- `GET /api/rules` — правила начисления очков; поддерживает фильтры `day` и `elementKey`.
- `POST /api/auth/register` — регистрация пользователя по имени пользователя, паролю и нику в игре.
- `POST /api/auth/login` — вход пользователя по имени пользователя и паролю.
- `GET /api/me/element-values` — сохраненные значения пользователя по элементам.
- `PUT /api/me/element-values` — обновление значений пользователя.
- `DELETE /api/me/element-values` — сброс значений пользователя.
- `GET /api/me/calculator` — состояние конструктора и текущий расчет.
- `PUT /api/me/plan-items` — сохранение выбранных ресурсов, запасов и плана траты.
- `PUT /api/me/speedup-allocation` — общий запас универсальных ускорений.
- `PUT /api/me/training-plan` — параметры казармы и расчет обучения.
- `DELETE /api/me/calculator` — полный сброс калькулятора.
- `POST /api/calculate` — расчет очков по ресурсам и коэффициентам.

Интерактивная документация API:

- `GET /api-docs` — Swagger UI;
- `GET /openapi.json` — OpenAPI 3.1 схема.

## База данных

Для MVP используется SQLite. Файл базы задается переменной `SQLITE_DATABASE_PATH`.

Backend при старте создает таблицы и наполняет справочники из кода:

- `elements` — игровые элементы, категории, единицы измерения, иконки и дни события;
- `event_rules` — действия, очки и дни, в которые правило активно.
- `users` — минимальные аккаунты: имя пользователя, ник в игре и хеш пароля;
- `user_element_values` — сохраненные пользователем значения по элементам справочника.
- `user_plan_items` — выбранные правила, запас, планируемая трата и день.
- `user_training_plans` — параметры очереди, гарнизона и ускорений обучения.

## Docker

`docker-compose.yml` поднимает production-схему из двух сервисов:

- `backend` — внутренний Express API на порту `3000` внутри docker-сети;
- `web` — проектный nginx на внешнем порту `8080`.

SQLite-файл хранится в volume `sqlite-data` по пути `/data/tails-survive.sqlite`.

Nginx собирает Vue-фронт в статические файлы, отдает `index.html` для SPA-маршрутов и проксирует backend-ручки:

- `/api/*`;
- `/api-docs/*`;
- `/openapi.json`;
- `/health`.

На VPS внешний nginx сервера можно настроить как TLS/front proxy до `web:80` или до опубликованного порта `8080`.
