# Цель продукта

## Основная идея

Калькулятор помогает игроку решить, стоит ли тратить накопленные ресурсы в ближайшую неделю Турбочерепашки или выгоднее пропустить событие и продолжить копить ради более высокого места в рейтинге.

Главный вопрос продукта:

> Сколько очков я могу получить сейчас, сколько получу после пропуска одной или нескольких недель и когда мне лучше идти в рейтинг?

## Целевая аудитория

Игроки, которые:

- копят ресурсы между неделями Турбочерепашки;
- намеренно пропускают отдельные недели;
- хотят заранее оценить результат траты ресурсов;
- ориентируются на конкретное место в рейтинге;
- хотят распределить ресурсы по дням события.

## Основные возможности

### 1. Сценарии «пушить или копить»

Пользователь вводит текущие запасы ресурсов и получает сравнение сценариев:

| Сценарий | Результат |
| --- | --- |
| Пушить сейчас | Сколько очков можно заработать на ближайшей неделе |
| Пропустить 1 неделю | Прогноз очков после дополнительной недели накопления |
| Пропустить 2 недели | Прогноз очков после двух дополнительных недель накопления |

Для каждого сценария показываются:

- общая сумма очков;
- ожидаемое место или достижение цели;
- самые сильные дни недели;
- вклад каждого типа ресурсов.

### 2. План траты ресурсов по дням

Калькулятор показывает доступные действия и ожидаемые очки отдельно для каждого дня события.

Пользователь должен видеть:

- какие ресурсы учитываются в конкретный день;
- сколько очков даст каждый ресурс;
- общую сумму очков за день;
- общую сумму за всю неделю.

Ресурсы, доступные в несколько дней, не должны автоматически учитываться несколько раз. Пользователь выбирает день траты или использует предложенное распределение.

### 3. Цель по рейтингу

Пользователь задает желаемое количество очков или ориентир для места:

- топ-100;
- топ-50;
- топ-20;
- топ-10;
- собственное значение очков.

Калькулятор сравнивает прогноз каждого сценария с целью и показывает:

- достигнута ли цель;
- сколько очков не хватает;
- какой минимальный сценарий позволяет достичь цели;
- сколько недель рекомендуется копить.

### 4. Статус достижения цели

Вместо сложной оптимизации покупок и добора ресурсов калькулятор показывает простой статус:

| Статус | Значение |
| --- | --- |
| Цель достигнута | Текущих или прогнозируемых ресурсов достаточно |
| Почти достигнута | До цели не хватает небольшого количества очков |
| Недостаточно ресурсов | Для цели нужно продолжать копить |

Статус дополнительно показывает точную разницу между прогнозом и целью.

## MVP

Первая полезная версия должна включать:

1. Справочник правил начисления очков по дням.
2. Ввод текущих запасов ресурсов.
3. Расчет очков по каждому ресурсу.
4. Расчет очков по дням.
5. Общий прогноз очков за неделю.
6. Ввод целевого количества очков.
7. Статус достижения цели.
8. Сравнение сценариев «пушить сейчас», «пропустить 1 неделю», «пропустить 2 недели».
9. Аккаунт по нику и паролю.
10. Сохранение введенных значений по элементам справочника.
11. Сброс всех сохраненных значений пользователя.

## Удобство использования

- После входа пользователь должен сразу видеть последние введенные значения.
- Возле каждого инпута нужна иконка элемента и категория, чтобы не искать ресурс по длинному тексту.
- Для каждого элемента полезно показывать дни, в которые он участвует, например `1, 2, 5, 7`.
- Сброс значений должен быть отдельной явной кнопкой с подтверждением.
- Пользователь добавляет в рабочую область только те ресурсы, которые у него есть.
- Для каждой строки отдельно задаются запас, планируемая трата и день использования.
- Универсальные ускорения хранятся общим пулом и распределяются между исследованиями, строительством и обучением.
- Распределение на обучение не дает прямых очков: очки считаются по количеству готовых солдат.

## UI-библиотека

Рекомендуемый вариант для frontend: `Nuxt UI` в режиме plain Vue/Vite.

Причины:

- работает не только с Nuxt, но и с обычными Vue-приложениями;
- дает готовые доступные компоненты на базе Tailwind CSS и Reka UI;
- выглядит современнее типичных enterprise-компонентов;
- хорошо подходит для интерактивного калькулятора с карточками, вкладками, формами, бейджами и таблицами;
- проще быстро собрать приятный продуктовый интерфейс, чем на низкоуровневом наборе компонентов.

Альтернативы:

- `shadcn-vue` — если хотим максимальный контроль над кодом компонентов и готовы больше собирать руками;
- `PrimeVue` — если понадобится много сложных таблиц и enterprise-компонентов;
- `daisyUI` — если нужен самый быстрый Tailwind-старт, но без полноценной Vue-компонентной модели.

## После MVP

- сохранение нескольких наборов запасов;
- история результатов прошлых событий;
- ориентиры рейтинга конкретного сервера;
- рекомендации на основе прошлых результатов пользователя.

## Пока не входит в продукт

- сложная оптимизация того, какие ресурсы нужно купить;
- рекомендации по ценности каждого ресурса;
- автоматический подбор покупок для закрытия разницы до цели;
- учет других игровых событий, если они напрямую не влияют на расчет Турбочерепашки.

## Критерий полезности

После ввода запасов и цели пользователь должен за несколько минут получить понятный ответ:

> Пушить ближайшую неделю или пропустить ее, сколько очков получится в каждом сценарии и достаточно ли этого для выбранной цели.
